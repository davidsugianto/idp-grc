# Implementation Plan: GitHub Actions Self-Hosted Runner Operator

**Branch**: `001-github-runner-operator` | **Date**: 2026-06-30 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/001-github-runner-operator/spec.md`

---

## Summary

Build `idp-grc`, a Kubernetes operator in Go that lets users declare GitHub Actions self-hosted
runners as `GitHubRunner` custom resources. The controller handles the full lifecycle:
fetching a registration token from the GitHub REST API, creating and configuring a runner Pod,
monitoring its health, and de-registering it from GitHub on CR deletion. The operator is
packaged as a container image and installed via a Kustomize bundle.

---

## Technical Context

**Language/Version**: Go 1.22

**Primary Dependencies**:
- `sigs.k8s.io/controller-runtime` v0.18.4 — reconciler framework, envtest, leader election
- `k8s.io/api`, `k8s.io/apimachinery`, `k8s.io/client-go` (pulled transitively by controller-runtime)
- `net/http` (stdlib) — GitHub REST API calls (no third-party GitHub SDK)
- `slog` (stdlib, Go 1.21+) — structured JSON logging

**Storage**: Kubernetes etcd via CR `.status` fields (runner ID, phase, conditions). No external DB.

**Testing**: `go test ./...`; controller-runtime `envtest` for integration tests (real API server, no nodes).

**Target Platform**: Linux Kubernetes cluster, version 1.26+.

**Project Type**: Kubernetes operator (kubebuilder v4.1.1 layout).

**Performance Goals**:
- Runner registered and online within 3 minutes of CR creation
- Runner de-registered and Pod removed within 1 minute of CR deletion
- Controller reconcile loop: no blocking calls; GitHub API calls time out in ≤10s

**Constraints**:
- kubebuilder v4 standard layout (see Project Structure below)
- Kubernetes 1.26+ for stable CRD and server-side apply support
- Container image: distroless or scratch base (minimal attack surface)
- No external database or message broker

**Scale/Scope**: Single operator per cluster managing multiple `GitHubRunner` CRs across
all namespaces. One CR = one runner Pod (1:1, no pools).
